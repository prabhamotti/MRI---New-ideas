I am under the impression that we are underusing MRI. If we can use proteum(hydrogen) in MRI, then it is possible to employ the same idea to other atoms also. This will open a new era in MRI.

Using this we can track particular chemical element. The below are some examples.

 

Medical Field:

1)Tracking the cancer:

       Cancer cells can move into blood stream and spread cancer in other organs. For example let us consider bone  cancer. If this cancer cells travelled through blood stream and deposited in stomach, then use MRI to trace only calcium ions.

       This will show the traces of calcium in the stomach, thereby we can say that the patient is on the way to get stomach cancer. Because body is having certain chemicals only in certain parts in high density. If the same chemical found in some other

       Part in high level is abnormal. By quantifying these chemicals by  MRI it is possible to tell the next target of the cancer. I am giving a small list for example.

 

       Male organ => Generally observes gold,silver,tin,(very very small amount of) mercury

       Blood,Bone marrow => Iron

       Kidney => Nitrogen related things,Sodium

       Liver => Metals

       Skin => Sulphur

       Etc

 

      So, if any of the above body part got cancer then trace out the corresponding chemical element, then we will come to know where will be the next cancer attack.

      It can also give a different perception of the tumor.

 

2)Finding Carcinogens(Finding cancer in early level):

      Researches show that some carcinogens are triggering cancer. For example lithium triggers lung cancer. So if there is a doubt between tumor and cancer in lungs region, then use MRI scan to trace lithium in that region. If it is there you can

Say it is a cancer. Many of the workers who work in Cement factory have more chances for lung cancer. So, their lungs can be  diagnosed for cancer by checking them for lithium traces using MRI. Like this there are specific carcinogens for

Specific areas. By tracing them out we can found the cancer in early stage.

 

3)Finding Bone density:

    Women’s bone density will reduce in their old age. We  can use MRI to find its density

 

4)Finding Nicotine Level in Lungs:

    It is possible to measure the nicotine in lungs using the proposed method. Thereby we can say whether the patient is within the safety limit or not.

 

5)Finding Fat Level in Body:

    Same as above except the measured content is fat.  Thereby we can say whether the patient is within the safety limit or not. This can be used as a measuring tool for liposuction.

Using the proposed method we can scan the rubber part after 5000 cycles and can say whether will it brake or not. This will reduce their testing time. This will detect the hiding defects in the rubber part after the 5000 cycles. Even after 10000 cycles the rubber part might have got some cracks in the inner part ,but not visible in the outer side. This can be detected by scan. This will ensure the quality of that automobile part. It is possible to make this testing in static condition or in dynamic condition.

Apart from the above ideas  I am having the following observation , and I thought that this can be applied for some other functional areas also. This is coinciding with the Deborah disanzo’s speech about some software tools are breaking. I couldn’t remember the date exactly.

 

Conflicting Approaches: Some of the codes I faced in Philips are “running code” not “impeccable code”.  Consider the following scenario. Manager is asking a programmer to complete a task. But that programmer has to make the code by consulting with

Architect or senior technical person. This technical person will try to make the programmer to implement “standard” as much as possible. When the coder makes the code according to the technical person, the manager asks for its progress and gets to know that the task is progressing. When the deadline approaches the programmer is still half the way to the completion because of implementing “standard” AMAP .  So, mostly he will try to complete the task rather than concerning “standard ways”. Because his completion is the only thing measured by the manager not the code’s standard.

In this way the conflicting approaches are effectively reducing the available time to the coder. In these kind of situations many persons will try to search for the quick solution. Third party library(open source projects) is one of the quickest solution. So, the coder will immediately use it and show his completion. This code will run perfectly at that time. When the time goes on the next person who works on that code wont remove that third party tool. That will cost extra time and unnecessary adversary effects. So, he will blindly follow what his fore runner followed. Here starts the blemish on code base. Open source projects are not long running projects. So, when the open source project developers stop their project then the impact will be on our code base, because now we are holding an non updated tool. Now to move forward in the compatibility driven environment we have to take care of all these kind of loose ends. Generally most of the people don’t care about open source project source code. They will compile it , then they will use the dll inside the project. Some people will directly download the dlls from open source project. When this dll makes problem, then solving it is tedious. If any other project incorporate this code, then this code will contaminate that project. Now the impact is doubled. So, more we use this code more we are contaminating the code base.

 

Using third party library(open source projects) is a common thing in coding. Sometimes it will come handy. But sometimes it will blemish the code base as explained above.

 

Result:

If a person works in an organization for a long time then most of his activities will be aligned towards the organization behavior. Based on this and considering the above scenario after some 1 or 2 year he will work to complete the tasks rather than understanding them. Because that is what he is asked and that is how he is measured. Thereby persons who are completing tasks on time without considering code blemishing will get good performance review. But, some persons who work hard to provide a good code have to take double work load. First they have to correct the old code(which has all those hidden errors), then they have to implement their task. These persons wont able to complete the task within the given time. Solution:

     Asking the programmer to apply the standard in all the way is OK. But when he sees that his performance is purely measured based on no of completions rather than his code standard, then he will slowly move from “implementing standard” to “completion on time”. But not all the task needs “standard”. For example one person just want to cross check another project by some test cases. Here these test cases are very urgent and they don’t need the “standard”. So, my suggestion is a task must be given with two parameters namely completion,standard. And performance must be calculated like (completion+standard)/10.  If the task is given with completion:8,standard:2. Then the coder will try his maximum on completion rather than “implementing standard”. If you give task with completion:2,standard:8. Then the coder will try to implement “standard” rather than completion. Here the management is explicitly motivating the coder in the required direction. The task will be completed with the desired quality and on time. The coder’s performance measurement is exact.

 

As an outcome all the employees can be categorized into below categories by measuring their task as explained above .

1 => Completion oriented persons. These persons are like quick solution. They can solve the problem immediately. They will take less time.

2 => Completion with standard. These persons are good for any work. They understood the requirement and also the standard. These persons can guide the above. These persons will take time slightly higher than the first category.

3 => Standard persons. Their work is guaranteed for long run. The above two categories work must be checked by these persons. This will ensure quality work. These persons will take good amount of time to complete a task.

 

And encourage good “SANITY TESTING” . It shouldn’t be a choice, it should be compulsory.

 

But, this kind of approach will take good amount of time. For this I am having the below solution.

 

Google Enhanced Code Base

1)   Google is providing a custom search option, which can be used according to our need. Using this we can search all the code base within Philips, and this behavior can be tailored as per the need. Outside persons cant access the code.

2)   Using this we can save the time. For example, in the past month I made a code for quantifying file size and folder size then I tested it thoroughly. It took me 3 days(24 working hours). If another person in Best needs the same functionality, then

        He again has to make all the logic, code and testing , again 3 days of work. But, if we apply this “Google search” approach, then all he has to do is search within the code base for the word “File size” or anything related with file size

        Then he will come to know that there is already one code(which was created by me) is having the required functionality. So, he can use my code by downloading the dll.

        Here we are reducing the 3 days’ work into 5 minute search.

3) The positive side effect is one code will be used where ever there is a need for it. So, even if one person couldn’t find a hiding bug in it, the other person may find it. The more persons use the same code is like extending the net to catch the fish(bug).

4)The accesses can be modified to protect the copyrighted algorithms.

5)Based on the point 2 , this approach will reduce the production time.

6)Quality will be increased. Because when 10 persons use the same code, then each one will see the same code in different point of view and they will make their suggestions, this will improve the quality.
